# Self-Training with Pseudo-Labeling Guide

## Overview

When faced with limited labeled data, **Self-Training** (also known as Pseudo-Labeling) allows the model to leverage large pools of unlabeled imagery to improve its generalization and boundary precision. 

In this repository, we implement a **Teacher-Student** framework:
1.  **The Teacher**: A frozen model (typically initialized from a previous best checkpoint) that generates predictions on unlabeled data.
2.  **The Student**: An active model that learns simultaneously from:
    -   **Ground Truth (GT)**: Real labels from the small labeled dataset.
    -   **Pseudo-Labels (PL)**: High-confidence predictions generated by the Teacher from the unlabeled dataset.

## Core Mechanisms

### 1. DualStreamDataset (`utils/dataset.py`)
To ensure the model sees both data types in every manual optimization step, the `DualStreamDataset` interleaves the streams:
-   It takes a list of labeled files and a directory of unlabeled files.
-   Each `__getitem__` call returns a tuple: `(labeled_img, labeled_mask, unlabeled_clean, unlabeled_aug)`.
-   **Weak Augmentation/Clean**: Provided to the Teacher to ensure the most stable pseudo-label generation.
-   **Strong Augmentation**: Provided to the Student for the same image, forcing it to learn features that remain invariant under heavy transformation.

### 2. Confidence Filtering
Not all Teacher guesses are correct. To prevent the Student from "learning noise," we apply a confidence threshold defined by `PSEUDO_LABEL_THRESHOLD` (default: `0.90`):
-   If the Teacher's highest class probability for a pixel is $> 0.90$, that pixel is assigned a Pseudo-Label.
-   If the probability is $\le 0.90$, the pixel is assigned the `IGNORE_INDEX` (`-1`).

### 3. Masked Loss Propagation
The `IGNORE_INDEX` ensures that uncertain pixels do not contribute any gradient during backpropagation. This allows the model to "focus" only on the parts of the unlabeled image where the Teacher is certain about the class (e.g., clear forest boundaries or distinct water bodies).

## Execution Guide

### 1. Configuration (`config/config.yaml`)

Enable the feature and point to your unlabeled data:

```yaml
# -----------------------------------------------------------------------------
# Self-Training / Pseudo-Labeling
# -----------------------------------------------------------------------------
SELF_TRAINING: true
UNLABELED_IMG_PATH: "/path/to/unlabeled/images"
PSEUDO_LABEL_THRESHOLD: 0.90
IGNORE_INDEX: -1
```

### 2. Running Training

Simply run the standard training script:

```bash
python train.py --config config/config.yaml
```

**What happens under the hood?**
1.  The system checks for a pre-existing checkpoint (e.g., `checkpoints/UNetPlusPlus_best.pth`) to initialize the **Teacher**.
2.  The **Student** is initialized using your configured `ENCODER_WEIGHTS` (e.g., ImageNet).
3.  The `DataLoader` automatically adjusts: if your `BATCH_SIZE` is 8, the system will load 4 labeled and 4 unlabeled pairs per step to maintain memory stability.
4.  Pseudo-labels are generated on-the-fly on the GPU.

## Best Practices

### The "Senior Developer" Principle
The Teacher should ideally be better than, or at least as good as, a random-init model. 
-   **Recommended**: First train a model using standard Supervised Learning or Transfer Learning (Stage 3). 
-   Use that successful checkpoint as the Teacher for a Self-Training run to further refine the results using the full unlabeled set.

### Augmentation Balance
-   **Teacher**: Use minimal or "weak" augmentation (flips, slight rotations). This yields the most accurate labels.
-   **Student**: Use "strong" augmentation (Color jitter, CoarseDropout, GridDistortion). This prevents the Student from simply memorizing the Teacher's output and forces it to understand semantic structure.

### Threshold Tuning
-   **High Threshold (0.95+)**: High precision, but low recall (fewer pseudo-labels used). Best when the Teacher is noisy.
-   **Lower Threshold (0.70 - 0.85)**: More pseudo-labels, but higher risk of confirming the Teacher's errors. Best when you have very few labels and need any signal you can get.

## Comparison: SSL vs. Self-Training

| Feature | Self-Supervised (MAE) | Self-Training (Pseudo-Label) |
| :--- | :--- | :--- |
| **Input** | Unlabeled Images | Unlabeled Images + GT Labels |
| **Output Type** | RGB Reconstruction | Semantic Map |
| **Learning Goal** | General Geometry & Texture | Class-Specific Boundaries |
| **Best Used As** | Pre-training Step (Phase 1) | Fine-tuning/Refining Step (Phase 4) |
